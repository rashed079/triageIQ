Workflow

How TriageIQ Works

A four-stage pipeline from patient symptom intake through clinician-verified sign-off.

01
💬

CTAS-Aligned Intake

Conversational AI chatbot guides patients through structured symptom collection — aligned with Canadian Triage and Acuity Scale (CTAS) criteria. Red-flag detection triggers immediate clinical escalation.

02
🧠

Structured Extraction

Claude API extracts CTAS fields from the conversation — chief complaint, onset, severity, duration, associated symptoms — into a structured JSON object for clinical review.

03
📚

Evidence-Grounded AI

RAG engine retrieves relevant clinical guidelines from a FAISS vector index and passes them to Claude alongside patient data — generating differential diagnoses with source citations.

04
✅

Clinician Verification

Clinician reviews AI-generated differentials, accepts or rejects each one, selects final CTAS level, and signs off. Full audit trail maintained. Zero autonomous AI decisions.

Key Features

Designed for Clinical Reality

🚨

Red-Flag Detection

Keyword and pattern matching immediately escalates life-threatening presentations (chest pain, stroke symptoms, respiratory distress) — bypassing the intake queue and alerting clinical staff instantly.

📋

CTAS-Aligned Intake

Conversational intake engine follows Canadian Triage and Acuity Scale criteria — collecting chief complaint, onset, severity (0–10), duration, and associated symptoms in a structured, auditable format.

🔬

Evidence-Grounded Differentials

RAG engine retrieves relevant clinical guidelines and passes them alongside patient data to Claude — generating differential diagnoses with source citations. No hallucinated evidence.

🔒

Full Clinician Oversight

Every AI output requires explicit clinician accept/reject review and manual CTAS selection before sign-off. No autonomous AI decisions. Complete audit trail per session.

🏥

Local-First Architecture

Designed to run 100% locally — no patient data leaves the clinical environment. Cloud deployment path targets AWS ca-central-1 for PHIPA (Ontario) data residency compliance.

📊

Session Audit Trail

Every intake turn, extraction result, AI differential, clinician decision, and sign-off is logged with timestamps. Full audit trail accessible via REST API for clinical governance.
